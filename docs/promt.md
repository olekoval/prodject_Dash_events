# Роль

Ти — Senior Data Analyst та розробник Dash/Plotly дашбордів з глибоким розумінням візуалізації медичних/фінансових даних.

# Контекст проєкту

Я розробляю аналітичний дашборд на Python (Dash/Plotly) з підключенням до PostgreSQL.

### Структура та файли проєкту:

* app.py: Основний файл додатка.
* home.py: Модуль дашборду (макет та логіка візуалізації).
* query\_home.py: Скрипт з SQL-запитами до БД.
* Результати запитів (клієнтські датасети): gr\_services\_all\_25\_26.parquet, gr\_services\_first\_letter\_25\_26.parquet, gr\_services\_Q\_25\_26.parquet.

### База даних:

Дані завантажуються з двох аналогічних таблиць: analytics.rds\_smd\_patient\_treatments\_2025 та analytics.rds\_smd\_patient\_treatments\_2026.

Схема таблиць:

* patient\_id (character varying) — унікальний ID пацієнта
* edrpou (character varying) — ЄДРПОУ закладу
* report\_month (numeric) — звітний місяць
* packet\_number (text) — номер пакета
* service\_number (text) — номер послуги
* is\_payed (boolean) — прапорець оплати
* is\_correct (boolean) — прапорець коректності/валідності
* emz\_list (ARRAY) — список ЕМЗ
* performer\_list (ARRAY) — список виконавців
* details (json) / tech\_details (json) — детальні/технічні дані
* comment\_text (text), hash\_id (uuid)

# Задача

Проаналізуй надану структуру даних та додані файли (app.py, home.py, query\_home.py — додані у вкладенні).

Запропонуй додаткові аналітичні показники (KPIs), графіки та фільтри, які варто включити в дашборд на додачу до вже існуючих.

# Вимоги до відповіді:

1. Картки KPI (Key Metrics): Які нові агреговані метрики (наприклад, % оплаченого, сер. чек на пацієнта тощо) варто винести в верхню панель?
2. Візуалізації (Графіки): Запропонуй 3–5 конкретних типів графіків Plotly (bar, line, treemap, boxplot тощо), вказавши:

   * Які саме поля зі схеми використовувати (осі X/Y, групування).
   * Яку бізнес-цінність/інсайт дає цей графік.
3. Порівняльний аналіз (YoY): Як найкраще візуалізувати динаміку між 2025 та 2026 роками?
4. Фільтри та інтерактивність: Які зрізи (Dropdowns, RangeSliders) варто додати для деталізації?
5. Оптимізація SQL (коротко): Якщо для нових графіків потрібні нові агрегації, запропонуй концепт оптимізованого SQL-запиту.
